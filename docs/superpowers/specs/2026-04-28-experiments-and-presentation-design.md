## 3. Сравнение оптимизаторов

**Файл:** `notebooks/02_train.ipynb` (новая секция)
**Что:** 4 оптимизатора — **Adam (baseline)**, SGD (momentum=0.9), RMSprop, Adagrad — обучаются по **10 эпох без EarlyStopping** на одних данных. Кривые val loss выводятся на одном графике. Adam идёт первым и выделяется как эталон (жирная линия / другой цвет).
**Как:**
- Модель пересоздаётся с нуля для каждого оптимизатора (одинаковый seed)
- lr одинаковый (3e-4) для честного сравнения

**Выход:** скриншот для слайда 8.

## 4. CrossEntropy vs MSE

**Файл:** `notebooks/02_train.ipynb` (новая секция)
**Что:** Параллельный эксперимент — две модели, одна с MSELoss (текущая), другая с BCELoss (Sigmoid вместо Tanh на выходе).
**Как:**
- `LaneCNNClassifier` — копия `LaneCNN` но с `Sigmoid` вместо `Tanh` на выходе, обучается с `BCELoss()`
- Метки для BCE: `(|angle| < 0.15).float()` → 1.0 = в полосе, 0.0 = вне
- Сравниваем: val_loss, accuracy (бинарная классификация)
- В агент/ПИД `LaneCNNClassifier` **не** идёт — только для сравнения
**Выход:** таблица метрик + скриншот для слайда 9.

## 5. Optuna vs Grid Search (подбор PID)

**Файл:** `notebooks/03_agent_pid.ipynb` (новая секция)
**Что:** Optuna запускает байесовскую оптимизацию Kp, Kd, Ki — минимизирует CTE на валидационной выборке. Сравнивается с уже имеющимся grid search.
**Как:**
- `optuna.create_study(direction="minimize")`
- Objective: создать PID с предложенными параметрами, прогнать по val-массиву ошибок, вернуть CTE
- n_trials=50
- Показать: кривую сходимости Optuna (лучший CTE по trial-ам) + итоговую таблицу Grid vs Optuna
**Выход:** скриншот для слайда 11.
